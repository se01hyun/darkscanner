# Dark-Scanner 중간 진행 보고서

**작성일**: 2026년 4월 6일  
**프로젝트명**: Dark-Scanner — 온라인 쇼핑몰 다크패턴 탐지 크롬 확장 프로그램

---

## 1. 프로젝트 개요

온라인 쇼핑몰에서 소비자의 구매 판단을 의도적으로 흐리거나 압박하는 UI 설계 기법을 **'다크패턴(Dark Pattern)'** 이라 합니다. Dark-Scanner는 이런 패턴을 사용자가 쇼핑 중 실시간으로 알아챌 수 있도록 경고를 띄워주는 크롬 브라우저 확장 프로그램입니다.

탐지 대상은 공정거래위원회 「온라인 다크패턴 자율관리 가이드라인」(2023)과 한국소비자원 「온라인 쇼핑몰 다크패턴 실태조사」(2023) 두 공신력 있는 자료를 기준으로 선정하였습니다. 실태조사에서 사용 빈도가 높은 패턴 6종을 우선순위 순서대로 구현합니다.

## 3. 완료된 작업 (Phase 0 ~ Phase 1)

### Phase 0 — 프로그램 기본 구조

크롬 확장 프로그램으로 로드할 수 있는 최소 파일 구조를 완성했습니다. 구글이 2024년부터 요구하는 최신 확장 프로그램 규격(Manifest V3)을 준수하여 작성하였습니다.

---

### Phase 1 — 압박형 다크패턴 탐지 3종

실태조사에서 가장 많이 발견된 순서대로 구현하였습니다.

#### P1-A. 다른 소비자의 활동 알림 (실태조사 빈도 1위, 93.4%)

> "지금 247명이 보는 중", "최근 50명이 구매했습니다", "장바구니에 12명이 담았습니다"

쇼핑몰에서 다른 사람들의 행동을 강조하여 구매를 서두르게 만드는 수법입니다. 실태조사에서 조사 대상 사이트의 93.4%에서 발견된 가장 보편적인 패턴입니다. 소비자가 "남들도 사니까 나도 사야겠다"는 심리(사회적 증거 효과)를 의도적으로 자극합니다.

**구현 방식**:

1. **텍스트 규칙 탐지**: "N명이 보는 중", "최근 N명 구매", "재고 N개 남음" 등 수십 가지 문구 변형을 하나의 규칙으로 정의하여 페이지 전체를 스캔합니다.
2. **동적 요소 감시**: 많은 쇼핑몰이 처음 페이지를 불러올 때는 해당 문구가 없다가 나중에 팝업·배너 형태로 삽입합니다. 이를 포착하기 위해 페이지 로드 이후에도 DOM 변화를 실시간으로 감시합니다.
3. **중복 방지**: 같은 요소를 여러 번 탐지하지 않도록, 한 번 탐지된 요소는 내부 목록에 기록하여 중복 경고를 차단합니다.

---

#### P1-B. 감정적 언어사용 (실태조사 빈도 2위, 86.8%)

> "혜택 포기하기", "그냥 비싸게 살게요", "무시하고 나가기"

팝업이나 가입 유도 화면에서 거절 버튼에 자기 비하적이거나 손해를 강조하는 문구를 넣어, 소비자가 심리적 불편함 때문에 클릭을 망설이게 만드는 수법입니다. '아니오'를 선택하는 것이 바보 같거나 손해를 보는 것처럼 느끼게 만드는 것이 핵심입니다.

이 패턴은 쇼핑몰마다 표현이 제각각이어서 단순한 금지어 목록으로는 탐지 범위에 한계가 있습니다. 예를 들어 "괜찮아요, 정가로 살게요"는 어떤 키워드 목록에도 걸리지 않지만 맥락상 동일한 압박 수법입니다.

**구현 방식**:

1. **버튼·링크 텍스트 수집**: 페이지 내 모든 버튼, 링크, 클릭 가능한 요소에서 텍스트를 추출합니다.
2. **Claude AI 일괄 분석**: 추출한 텍스트 전체를 Claude AI(Anthropic)에 한 번에 전송하여 각 문구가 감정적 압박을 담고 있는지 분류받습니다. AI를 사용함으로써 문구 변형이나 우회 표현도 잡아낼 수 있습니다.
3. **비용 최적화**: 요소 하나당 API를 한 번씩 호출하면 수십~수백 번의 요청이 발생합니다. 이를 방지하기 위해 페이지당 최대 50개를 묶어 1회만 요청합니다.
4. **판단 근거 전달**: AI는 탐지 결과와 함께 "왜 이 문구가 감정적 언어인지" 이유도 반환합니다. 이 정보는 이후 사용자에게 보여줄 툴팁 설명에 활용됩니다.

---

#### P1-C. 시간제한 알림 (실태조사 빈도 3위, 75.0%)

> "오늘만 할인", "마감 임박", "타임세일 2:47 남음"

제한 시간을 표시하여 소비자가 충분히 생각할 여유 없이 빠르게 결정하도록 압박하는 수법입니다. 문제는 이 카운트다운이 진짜 마감 시간이 아닌 경우가 많다는 점입니다. 사이트에 다시 접속하면 타이머가 리셋되거나, 사람마다 다른 시간이 표시되는 사례가 실태조사에서 다수 확인되었습니다.

**구현 방식**:

1. **텍스트 패턴 탐지**: 시간 압박 문구를 아래 네 가지 유형으로 분류하여 탐지합니다.

   | 유형 | 탐지 문구 예시 |
   |---|---|
   | 오늘 한정 | "오늘만", "오늘 한정", "오늘까지", "지금만", "이 가격 마지막" |
   | 마감·재고 강조 | "마감 임박", "한정 판매", "한정 수량", "자정까지", "세일 마감 임박" |
   | 숫자 포함 시간 표현 | "3시간 남음", "20분 이내", "D-2", "3일 후 종료", "오늘 오후 6시 마감" |
   | 영문 | "flash sale", "ends in 3" |

   단순 단어 일치가 아닌 규칙(정규식) 기반이라, "3 시간  남았어요"처럼 띄어쓰기가 다르거나 숫자가 바뀌어도 탐지합니다.
2. **타이머 변화 감지**: `02:47` 같은 디지털 시계 형식의 숫자가 실제로 줄어드는지 실시간으로 확인합니다. 숫자 형식으로 표시되어 있어도 변하지 않으면 타이머로 취급하지 않습니다.
3. **진위 판별 (이 프로젝트의 핵심 차별점)**:

타이머가 탐지되면 그 숫자가 어디서 왔는지 출처를 추적합니다.

| 판정 | 의미 | 판별 방법 |
|---|---|---|
| **진짜 (REAL)** | 서버에서 실제 마감 시간을 받아 표시 | 페이지 코드 내에 서버 발행 시간 데이터가 명시되어 있음 |
| **가짜 (FAKE)** | 브라우저가 자체적으로 만들어낸 숫자 | 서버 데이터는 없고, 브라우저 내부 저장소에만 타이머 값이 존재 (사이트에 다시 들어오면 리셋됨) |
| **불명 (UNKNOWN)** | 출처를 확인할 수 없음 | 코드로만 구동되며 외부 근거 없음 |

단순히 "타이머가 있다"고 알려주는 데 그치지 않고, 그 타이머가 믿을 수 있는 정보인지 소비자에게 근거와 함께 알려주는 것이 이 기능의 목표입니다.

## 5. 기술적 특이사항

- **AI 활용**: 단순 키워드 매칭만으로는 탐지 불가능한 감정적 언어 변형을 Claude AI NLP로 처리
- **진위 판별**: 시간제한 알림이 '실제 마감'인지 '조작된 가짜 타이머'인지 분석하는 로직 — 단순 탐지를 넘어 소비자에게 더 정확한 정보 제공 가능
- **실시간 감시**: 쇼핑몰이 나중에 끼워 넣는 팝업·배너도 즉시 감지 (MutationObserver)
- **공식 기준 근거**: 탐지 대상 선정, 우선순위, 설명 문구 모두 공정거래위원회·한국소비자원 자료 기반
