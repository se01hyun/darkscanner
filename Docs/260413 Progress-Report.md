## 2. 완료된 작업 (1주차 보고서 이후)

### 2-1. Phase 3 — 경고 UI 구현

탐지 결과를 사용자에게 시각적으로 전달하는 UI 레이어를 전부 구현했습니다.

#### (1) 심각도별 하이라이트 오버레이

탐지된 DOM 요소 위에 색상 테두리를 오버레이로 주입합니다.

| 등급 | 표시 방식 | 적용 대상 |
|---|---|---|
| 확정-높음 | 빨간 실선 (`#ff453a`) | 감정적 언어사용, 소셜 프루프 |
| 확정-보통 | 주황 실선 (`#ff9f0a`) | 시간제한 알림 (REAL 판정) |
| 의심 | 주황 점선 | 시간제한 알림 (UNKNOWN 판정) |

스크롤·리사이즈 시에도 오버레이 위치가 탐지 요소를 정확히 따라가도록 구현했습니다.

#### (2) 호버 툴팁

탐지 요소에 마우스를 올리면 다음 정보가 담긴 툴팁이 나타납니다.

- 패턴 유형 (예: "압박형 — 감정적 언어사용")
- 탐지된 텍스트
- 탐지 근거 (왜 이 문구가 다크패턴인지 한 줄 설명)
- 신뢰도 / 심각도 태그
- 카운트다운의 경우 진위 판별 결과 (REAL / FAKE / UNKNOWN)

#### (3) 확장 프로그램 아이콘 배지

페이지 내 탐지 건수를 크롬 툴바의 확장 프로그램 아이콘 배지로 실시간 표시합니다. (숫자가 없는 경우 배지 자동 제거)

### 2-2. 감정적 언어 탐지 방식 전환: Claude API → 규칙 기반

1주차 보고서에서는 감정적 언어(P1-B)를 Claude AI API로 분류하는 방식을 채택했었습니다. 이번 주 실제 구현·테스트 과정에서 다음 문제가 확인되어 **규칙 기반 정규식 패턴 매칭**으로 전환했습니다.

**전환 이유**
- API 키 배포 문제: 확장 프로그램 패키지에 API 키를 포함할 수 없어 사용자가 직접 키를 입력해야 하는 구조가 되어 UX가 크게 저하됨
- API 쿼터 불안정성: 페이지당 수십 건의 버튼 텍스트가 처리 대상이 되는데, 트래픽이 몰리면 429 오류가 발생하여 탐지 자체가 중단됨
- 오프라인 동작 필요: 쇼핑 환경에서 네트워크 지연이 탐지 결과 표시를 지연시키는 것은 허용 불가

**현재 구현 방식**

4개 카테고리 33개 정규식 규칙으로 탐지합니다.

| 카테고리 | 규칙 예시 |
|---|---|
| 혜택·기회 포기 프레이밍 | "포기하기", "혜택 포기", "쿠폰 없이" |
| 자기비하적 거절 | "비싸게 살게요", "손해 봐도 괜찮아요", "정가로 살게" |
| 무시·경시 프레이밍 | "무시하고 나가기", "그냥 닫기", "신경 안 쓸게요" |
| 직접적 손실 인정 | "안 받겠습니다", "혜택 없이 계속", "나중에 할게요" |

각 규칙에는 **탐지 근거 문자열**(예: "거절을 '포기'로 표현하여 손실감 유발")이 포함되어 있으며, 이 내용이 툴팁에 바로 표시됩니다. API 응답을 기다릴 필요 없이 오프라인으로 즉시 동작합니다.

### 2-3. 팝업 UI 전면 리디자인

확장 프로그램 팝업을 macOS 다크 모드 디자인 언어 기준으로 재설계했습니다.

**주요 변경사항**

| 항목 | 이전 | 현재 |
|---|---|---|
| 색상 테마 | 없음 (기본 흰 배경) | macOS 다크 모드 (`#121212` 배경) |
| 탐지 현황 표시 | 단순 텍스트 목록 | 위험도 점수 (0–100) + 프로그레스 바 + 태그 |
| 탐지 카드 | 없음 | 글래스모피즘 카드, 좌측 컬러 바로 심각도 구분 |
| 폰트 | 기본 | Pretendard (가독성 최적화) |

위험도는 탐지 건수 및 심각도 가중합으로 0–100 점수로 환산되며, "안전 / 주의 / 경고 / 위험" 4단계 태그로 표시됩니다.

### 2-4. 탐지 품질 개선 (버그 수정)

| 커밋 | 내용 |
|---|---|
| 소셜 프루프 오탐 수정 | 정규식 개선 — 숫자 없는 일반 문장 오탐 차단 |
| 카운트다운 오탐 수정 | 시간 형식이 아닌 일반 숫자 표기 오탐 차단 |
| 부모-자식 중복 탐지 방지 (×2) | 같은 패턴이 상위·하위 요소에 모두 매칭될 때 상위 1건만 탐지 |
| IIFE 격리 | `detectCountdown`, `detectSocialProof` 함수명 전역 충돌로 소셜 프루프가 탐지되지 않던 문제 수정 |

### 2-5. 테스트 케이스 추가

Jest 기반 단위 테스트를 보강했습니다.

- 각 탐지기(소셜 프루프, 감정적 언어, 카운트다운)의 정탐 케이스
- 오탐 회귀 방지 케이스 (수정된 버그가 재발하지 않도록 고정)

## 5. 기술적 주요 결정 사항 (이번 주 추가)

| 결정 | 내용 | 이유 |
|---|---|---|
| 감정적 언어 탐지 오프라인 전환 | Claude API 제거, 정규식 33개 규칙으로 대체 | API 키 배포 불가, 쿼터 불안정, 지연 허용 불가 |
| 오버레이 위치 동기화 | `ResizeObserver` + `scroll` 이벤트로 오버레이 실시간 재계산 | 쇼핑몰은 스크롤 시 레이아웃이 변하는 경우가 많아 정적 좌표로는 불가 |
| 팝업 위험도 점수 | 건수 × 가중치 합산 후 0–100 스케일 | 단순 건수보다 심각도 반영 정보 제공 가능 |
